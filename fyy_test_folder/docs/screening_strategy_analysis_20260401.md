## 一、回测系统 Bug 诊断

### 1.1 问题：并行分析导致交易时间穿越

**发现依据**：读取 `fyy_test_folder/backtest/results/` 下所有回测 JSON，逐个检查 `results[].date` 字段顺序和 `portfolio.trades[].date` 字段顺序。

**数据证据**：

| 文件 | results 按日期排序 | trades 存在倒序 |
|------|-------------------|----------------|
| backtest_688016_20260401_224056.json | 否 | **是** (03-16 买入 → 03-09 卖出) |
| backtest_688013_20260401_195855.json | 否 | **是** |
| backtest_688578_20260401_140659.json | 否 | **是** |
| backtest_601318_2026Q1_wedfri.json | 是 | 否 (此文件未受影响) |
| 其余 15 个文件 | 否 | 否 (但 results 顺序仍为乱序) |

**根因**：`run_backtest.py` 使用 `ThreadPoolExecutor` 并行调用 LLM 分析，结果按 `as_completed()` 完成顺序 append 到 `self.results`。而 `simulate_portfolio()` 直接遍历 `self.results` 模拟交易，未先按日期排序。

**影响**：
- `688016` 原始报告收益 +16.97%，按正确日期重排后实际约 +9.48%（虚增 7.49 个百分点）
- 部分文件交易序列出现"先买后日期才卖前日期"的逻辑错误

**修复**（已完成 2026-04-01）：
- 在 `simulate_portfolio()` 入口处对 `self.results` 按 `date` 字段排序
- 在 `run()` 方法中组合模拟前也做一次排序
- 统一报告 schema，确保 `portfolio` 字段名一致

### 1.2 问题：结果文件 schema 不一致

**数据证据**：

| 文件来源 | 组合模拟字段名 | 包含 trades 明细 |
|----------|---------------|-----------------|
| 早期手动文件 (2026Q1_wedfri) | `backtest_summary` | 否 (只有 trades 计数) |
| 自动回测文件 | `portfolio` | 是 (完整交易列表) |

**影响**：批量横向比较时，需要同时兼容两种字段名，容易遗漏或误读。

## 二、四象限选股策略缺陷分析

### 2.1 分析对象

`scripts/stock_quadrant_filter.py` 及其输出 `output/stock_quadrant_filter.csv`。

该策略将股票分为四个象限：
- **象限 1**：强趋势牛股（年线向上、接近高点、中低波动）
- **象限 2**：强趋势熊股（排除）
- **象限 3**：高波动成长（宽幅震荡、高波动）
- **象限 4**：题材妖股（急涨急跌、极高波动）

### 2.2 输出结果

CSV 运行日期约为 2026-03-31，**仅输出 5 只股票，全部为象限 3（北交所）**：

| 代码 | 名称 | 象限 | 20日收益 | 60日收益 | 年度高点比 | 基本面评分 | 趋势评分 |
|------|------|------|----------|----------|-----------|-----------|----------|
| 920418 | 苏轴股份 | Q3 | -13.3% | -23.3% | 61.3% | 93.0 | 0 |
| 920098 | 科隆新材 | Q3 | -11.1% | -18.7% | 68.2% | 80.0 | 0 |
| 920885 | 星辰科技 | Q3 | -16.8% | -24.8% | 61.1% | 77.0 | 1 |
| 920169 | 七丰精工 | Q3 | -20.2% | -11.3% | 55.4% | 75.5 | 0 |
| 920665 | 科强股份 | Q3 | -11.2% | -22.8% | 56.3% | 73.0 | 0 |

**象限 1 结果：0 只。象限 4 结果：0 只。**

### 2.3 五个致命缺陷

#### 缺陷 1：Q3 不过滤趋势方向 — 选到的全是"高波动下跌股"

**代码依据** (`classify_quadrant` 方法, 约 L308-L320)：

```python
q3_conditions = [
    volatility > 0.03,                                              # 波动率 > 3%
    volatility <= 0.06,                                             # 波动率 <= 6%
    not (high_new_high and close_above_ma250 and ma250_slope > 0.001),  # 非强趋势
    not low_new_low,                                                # 非强下跌
]
# 只要 2 个条件满足即归为 Q3
```

Q3 的入选条件**完全不检查价格是在涨还是在跌**，只看波动率落在 3%~6% 区间。

**运行验证** (2026-04-01 在远端对 920885 和 920098 执行 `calculate_indicators`)：

| 指标 | 920885 | 920098 |
|------|--------|--------|
| close_above_ma250 | **False** | **False** |
| low_new_low | **True** | **True** |
| return_20d | **-17.1%** | **-10.9%** |
| return_60d | **-21.4%** | **-17.3%** |
| annual_high_ratio | 61.1% | 68.2% |
| trend_score | 1 | 0 |

两只票都在**创年内新低、远离年线、持续暴跌**，但因为 `low_new_low` 检查在 Q3 条件里是 `not low_new_low`（第 4 个条件），只要前 2 个波动率条件满足，`sum >= 2` 即可入选。

**结论**：Q3 名义上筛"高波动成长"，实际筛到的是"高波动暴跌股"。

#### 缺陷 2：Q1 条件过严 + 数据不足 — 0 结果

**代码依据** (`classify_quadrant` 方法, 约 L290-L302)：

```python
q1_conditions = [
    ma250_slope > 0.0005,       # 年线斜率明显向上
    close_above_ma250,          # 价格在年线上方
    volatility <= 0.04,         # 中低波动
    high_new_high or ratio > 0.85,  # 接近高点
]
# 需要 >= 3 个条件满足 AND trend_score >= 3
```

**数据依据**：`calculate_indicators` 要求 `len(df) >= 250` 才返回指标。但实际验证：

| 股票 | 实际行情天数 | 能否计算指标 |
|------|------------|------------|
| 688016 (心脉医疗) | 243 | **否** (< 250, 返回空) |
| 601318 (中国平安) | 243 | **否** (< 250, 返回空) |
| 920885 (星辰科技) | 1389 | 是 |

`stock_basic_info` 共 5501 条记录，但大量非北交所股票在 `stock_daily_quotes` 中通过 `period: "daily"` 查询时，由于数据入库时间较晚或数据覆盖不完整，行情天数不足 250 天。这导致 Q1 的候选池极度缩小，最终 0 结果。

#### 缺陷 3：无流动性门槛 — 北交所微盘股混入

**数据依据**：CSV 中市值列显示 `3.719e-08`（亿），这是因为 `stock_basic_info.total_mv` 字段单位是**万元**（如 601318 的 `total_mv: 10200.03` 表示 10200 万 = 102 亿），但代码在 `print_results` 里做了 `r.total_mv/1e9` 的除法，等于把万元当元再除以 10 亿。

实际上这 5 只北交所票市值约为 2~4 亿元，日均成交额估计在几百万元级别。没有任何成交额或市值下限过滤。

#### 缺陷 4：基本面评分虚高且无区分度

920418 苏轴股份获得 93 分（5 只中最高），但这只票 20 日跌了 13.3%、60 日跌了 23.3%。

**原因分析**：评分函数 `calculate_fundamental_score` 使用固定阈值绝对打分：

| 因子 | 权重 | 920418 实际值 | 得分 |
|------|------|-------------|------|
| ROE | 35% | 13.9% → >10 | 80 × 0.35 = 28 |
| 净利率 | 20% | N/A | 0 |
| 流动比率 | 15% | 很高 (低负债) | 100 × 0.15 = 15 |
| 毛利率 | 15% | ~30% | 100 × 0.15 = 15 |
| 负债率 | 15% | 14.9% → <30 | 100 × 0.15 = 15 |

总分 ≈ 73 + 20(净利率估算) = 93。问题在于：
- 不考虑 PE/PB 估值合理性
- 不考虑盈利增长方向
- 低负债的微盘股天然在这套评分里占优

#### 缺陷 5：离散分桶无法精确排名

四象限把股票分成 Q1/Q3/Q4/其他四个离散类别，桶内只能按基本面分粗排。一只"差一点没达到 Q1 门槛"的票和"完全不沾边"的票被同等丢弃。这种设计无法在数千只票中做 Top N 精确截取。
